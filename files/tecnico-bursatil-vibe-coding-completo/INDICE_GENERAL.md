🎓 FILOSOFÍA: POR QUÉ ESTA ESTRUCTURA
═════════════════════════════════════════════════════════════════════════════════

1. VIBE CODING, NO TIMELINE
   → No es "Semana 1, 2, 3". Es "Proceso 1, Proceso 2, Proceso 3"
   → Cada proceso tiene su bloque negro, independiente
   → Más flexible, menos presión

2. BLOQUES NEGROS, NO DOCUMENTOS TÉCNICOS
   → ChatGPT → Bloque negro → Claude Code/Antigravity
   → Sin redactar "documentación de código"
   → Ahorro de tokens, velocidad máxima

3. ETAPAS POR PROCESOS, NO POR Timeline
   → Backend completo → Frontend completo → Deploy
   → No "en paralelo parcial", es "secuencial pero rápido"
   → Cada etapa tiene clarity total

4. LÉEME.md PARA HUMANOS
   → No devs entienden jerga técnica
   → LÉEME explica qué pasa sin código
   → Tori puede leer sin dolor de cabeza

5. MCP + SKILLS = SUPERPODERES
   → Stitch ya configurado
   → Agregar conexiones = un archivo
   → Templates + librerías = reutilizable
